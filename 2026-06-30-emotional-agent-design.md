# 情感沟通 Agent 设计规格

> 日期：2026-06-30 | 状态：已确认，待实施

## 1. 概述

一个基于 LLM 的情感沟通对话 Agent，以单文件 HTML 形式交付。用户可在浏览器中与 Agent 对话，获取情感关系方面的共情倾听、分析和实用建议。

**核心原则**：温暖但不煽情 · 专业但不说教 · 支持但不替代专业心理咨询。

---

## 2. 技术选型

| 项 | 选择 | 说明 |
|----|------|------|
| 交付形式 | 单文件 HTML | CSS/JS 全部内联，`file://` 打开即用 |
| LLM API | 豆包（火山引擎） | 免费额度覆盖个人使用，OpenAI 兼容格式 |
| 可切换 | DeepSeek 等 | 改 baseURL + apiKey 即可切换 |
| 存储 | localStorage + JSON 导出 | 双重保障，见 §6 |
| 框架 | 无 | 纯原生 JS，零依赖 |

---

## 3. 架构

单文件内按模块划分：

```
emotional-agent.html
├── <style>        — 深色紫红主题（#08090d / #b98eff / #e85d75）
├── <div id="app"> — DOM 结构
└── <script>
    ├── SettingsManager   — API Key、baseURL、模型、温度参数
    ├── StorageManager    — localStorage 读写 + JSON 导出/导入
    ├── SafetyGuard       — 领域预检 + 危机关键词检测
    ├── ContextCompressor — 超出20轮时压缩早期对话为摘要
    ├── DeepSeekClient    — API 调用、流式输出（SSE）
    ├── ChatManager       — 会话 CRUD、上下文窗口管理、消息流控
    └── UIController      — DOM 渲染、消息列表、侧边栏、设置面板
```

## 4. System Prompt 设计

### 4.1 角色定义

```
你是"小暖"，一个专业的情感沟通助手。你帮助用户理解情绪、
改善沟通、经营关系。你不是心理医生，不提供医疗诊断。
```

### 4.2 对话原则

1. **先共情后分析** — 永远先接住情绪，再谈问题
2. **多问少断** — 用提问引导用户自己思考，不急于下结论
3. **具体化建议** — 给出可执行的话术和行为建议，不说空话
4. **中立不评判** — 帮助理解各方立场，不站队

### 4.3 领域边界（硬约束）

只回答情感沟通相关问题。允许范围：
- 恋爱关系、婚姻家庭、人际沟通、情绪管理、自我认知

拒绝范围及固定回复模板：
```
"抱歉，我是情感沟通助手，只能帮你解答感情和人际关系方面的问题。
 如果有情感方面的困扰，随时找我聊聊。"
```

拒绝触发：技术/编程、数学/科学、政治/宗教、其他与情感沟通无关的话题。

### 4.4 安全护栏

| 场景 | 响应策略 |
|------|---------|
| 自杀/自伤意念 | 表达关心 → 明确能力边界 → 提供专业热线 → 不深入探讨 |
| 家暴/虐待 | 提供求助资源，不建议自行处理 |
| 心理疾病症状 | 明确非医疗建议，建议就医 |
| 未成年人特殊话题 | 注意监护人知情权 |
| 操控/PUA 识别 | 主动识别并温和提醒 |

## 5. 界面布局

三栏结构：

```
┌──────────────┬─────────────────────┬─────────────┐
│  侧边栏 240px │    主对话区 flex:1   │  设置面板     │
│              │                     │  (抽屉式)    │
│  ＋ 新对话    │  消息气泡列表        │  API Key     │
│  💬 会话列表  │  (用户右对齐紫色底)   │  baseURL     │
│  🗑 清空全部  │  (Agent 左对齐暗底)   │  模型选择    │
│              │  输入框 + 发送按钮    │  温度参数    │
│              │                     │  导出/导入   │
└──────────────┴─────────────────────┴─────────────┘
```

**交互细节**：
- 侧边栏会话支持双击重命名、右键/× 删除
- Agent 回复流式逐字显示，打字时带闪烁光标
- 拒绝消息用淡红色边框卡片（`.reject-card`）与正常消息区分
- 快捷键：`Ctrl+N` 新对话、`Ctrl+,` 设置、`Esc` 关闭面板
- 消息时间戳显示在气泡下方，格式 `HH:mm`

**视觉风格**：
- 背景 `#08090d`，主色 `#b98eff`，强调色 `#e85d75`
- 文本 `#f6f2e8`，次级文本 `#aaa59a`
- 卡片圆角 8px，边框 `rgba(255,255,255,0.06)`
- 字体 `"Microsoft YaHei", "PingFang SC", system-ui`

## 6. 数据流与存储

### 6.1 消息发送流程

```
用户输入
  → UIController（渲染用户气泡 + 输入状态指示器）
  → SafetyGuard.领域预检（关键词匹配）
     → 命中非情感领域 → 本地拒绝（0 token），渲染拒绝卡片
     → 未命中 → 继续
  → ChatManager.buildContext（System Prompt + 压缩摘要 + 最近20轮）
  → DeepSeekClient.streamChat（POST /chat/completions, stream: true）
  → SSE 流式追加到 Agent 气泡
  → 完成后移除输入状态，show timestamp
  → StorageManager.save（localStorage）
```

### 6.2 上下文窗口管理

| 条件 | 策略 |
|------|------|
| ≤ 20 轮 | 全部带上，不做处理 |
| > 20 轮 | 超出部分调用 API 压缩为 ~200 token 摘要，注入 System Prompt 末尾 |
| 摘要后继续累积 | 直到再次超出 20 轮，将旧摘要 + 中间轮次重新压缩 |

摘要格式：`[历史背景：用户与伴侣因XX频繁争吵，已讨论过A/B/C方案，用户倾向于...]`

### 6.3 存储方案

| 层 | 键/文件 | 内容 |
|----|---------|------|
| localStorage | `emotion-agent:chats` | 所有会话完整 JSON |
| localStorage | `emotion-agent:settings` | API Key、baseURL、模型、温度 |
| JSON 导出 | 用户手动触发下载 | 完整数据备份，含会话 + 设置 |
| JSON 导入 | 用户选择文件上传 | 合并到现有数据（按 ID 去重） |

### 6.4 数据结构

```javascript
// 会话
{
  id: "uuid",
  title: "和伴侣频繁吵架",
  createdAt: "2026-06-30T10:00:00",
  updatedAt: "2026-06-30T12:30:00",
  messages: [
    { role: "user", content: "...", timestamp: "..." },
    { role: "assistant", content: "...", timestamp: "..." },
  ],
  summary: null | "历史背景摘要文本",
}

// 设置
{
  apiKey: "sk-...",
  baseURL: "https://ark.cn-beijing.volces.com/api/v3",
  model: "doubao-lite-128k",
  temperature: 0.7,
}
```

## 7. 错误处理

| 场景 | 处理 |
|------|------|
| 网络断开 | 红色边框气泡 + "网络连接失败，请检查网络后重试" + 重发按钮 |
| API Key 无效（401） | 弹出设置面板，提示并自动聚焦 Key 输入框 |
| API 429/500 | 显示错误信息 + "X 秒后自动重试"倒计时，最多 3 次 |
| 请求超时（>30s） | 显示 "响应超时，正在重试...（1/3）"，自动重试 |
| 流式中断 | 保留已接收内容 + 末尾标灰色 `[响应中断]`，可点"继续"补全 |
| localStorage 满 | 提示导出旧对话后清理，引导删除最早会话 |
| JSON 导入格式错误 | Toast："文件格式不正确，请选择有效的对话备份文件" |
| 未预期 JS 异常 | `window.onerror` 捕获 → 显示 "出了点问题，刷新页面即可恢复" |

## 8. 非功能需求

- **首次加载**：< 1 秒（单文件无网络依赖）
- **消息发送到首字显示**：< 2 秒（流式 SSE）
- **localStorage 读写**：同步操作，< 50ms
- **兼容性**：Chrome/Edge/Firefox 最近 3 个大版本
- **隐私**：API Key 存 localStorage，不出浏览器；对话数据不上传任何第三方

## 9. 范围边界

**Phase 1（本次）包含**：
- 单文件 HTML，基础对话功能
- 多会话管理（新建/切换/删除/重命名）
- 流式输出
- 领域预检拒绝
- 安全护栏
- 上下文压缩（>20 轮）
- 设置面板（API Key/baseURL/模型/温度）
- JSON 导出/导入

**Phase 1 不包含**：
- 用户登录/注册
- 后端服务/数据库
- Markdown 渲染（纯文本 + 换行）
- 多语言
- 语音输入
- 移动端原生适配（响应式基础适配即可）
