# Warmth 回复人性化优化 — 设计规格

> 日期：2026-07-14  
> 目标：从 Prompt 层、前端表现层、情感追踪层三方面让 AI 回复更自然、更有人情味  
> 约束：纯前端单文件 HTML，不改架构，不引入后端依赖

---

## 一、背景

当前 Warmth 的"损友嘴替"人设已经比较口语化，但存在三个问题：

1. **回复节奏机械** — 每次都秒回、每段都匀速打字，缺少真人"想一想再说"的感觉
2. **表达方式单一** — 模型容易形成"吐槽→分析→建议"固定模板，"说人话就是"等口头禅重复率过高
3. **情感共鸣不够** — 用户情绪强烈时仍然直接给建议，缺少"先接住情绪再谈事"的温暖感

## 二、设计目标

- 回复内容更有多样性，打破固定模板
- 回复节奏更接近真人聊天（有停顿、有变速、有"正在思考"的心理暗示）
- 模型能感知用户当前情绪并调整回复基调
- 所有改动在现有架构内完成，不引入新依赖

## 三、模块一：System Prompt 重写

### 3.1 新增"回复结构变异"指令

```
不要每次都按"吐槽→分析→建议"的模板回复。
偶尔直接给建议，偶尔先反问再分析，偶尔只共情不给建议。
连续两条回复的结构不能相同。
```

### 3.2 新增"情感共情前置"规则

```
对方情绪明显时（难过/愤怒/焦虑/委屈），第一句话必须先接住情绪。
参考句式："好难受啊这种感觉""气死了我能理解""换我我也烦"
禁止跳过情绪直接开始分析。情绪没被接住就开始给建议 = 机器人。
```

### 3.3 口头禅调整

**移除：** `说人话就是...`（重复率过高）

**替换方案：**

| 场景 | 口头禅 |
|------|--------|
| 吐槽离谱时 | 闹麻了 / 麻闹了 / 绷不住了 / 离谱 |
| 泼冷水时 | 别搞 / 别搞了 / 差不多得了 |
| 共鸣时 | 包的 / 确实 / 我懂 |
| 过渡句式 | 说白了就是... / 你想想... / 我问你... |

### 3.4 新增"避免重复"约束

```
同一场对话中，同一个口头禅最多用2次，同一个句式不要连续使用。
如果上一句用了"包的"，下一句共鸣时换"确实"或"我懂"。
```

## 四、模块二：前端表现层优化

### 4.1 思考动画升级

**现状：** 三个跳动的圆点  
**改为：** 随机思考文案 + 三圆点，每 1.5 秒切换

```
随机池：
"正在理解你的感受…"
"让我想想…"
"在打字了…"
"嗯，我想说…"
"让我组织一下语言…"
"这事儿得好好想想…"
```

**实现位置：** `_createTypingIndicator()` 方法

### 4.2 回复前随机停顿

流式响应结束后，打字机动画开始前，插入 300-900ms 随机延迟。  
模拟"看完消息 → 思考 → 开始打字"的自然间隔。

**实现位置：** `_send()` 中 `onAgentSegments` 回调，在 `typing.remove()` 之后、第一段打字机开始之前

### 4.3 打字速度三段式变速

不再匀速，每条消息内部按比例变速：

| 阶段 | 占比 | 速度倍率 | 效果 |
|------|------|---------|------|
| 开头 | 前 20% | ×1.3 | 慢，模拟斟酌措辞 |
| 中间 | 60% | ×1.0 | 正常速度，流畅输出 |
| 结尾 | 后 20% | ×1.3 | 慢，模拟收尾确认 |

短消息（<30字）不减速，保持灵动。

**实现位置：** `_send()` 中打字机循环逻辑，替换当前的固定 `speed` 参数

### 4.4 段间停顿动态化

不再固定 280ms，根据下一段长度随机波动：

| 段落长度 | 停顿范围 |
|----------|---------|
| < 80 字 | 200-400ms |
| 80-200 字 | 350-600ms |
| > 200 字 | 500-900ms |

**实现位置：** `_send()` 中 `segmentGap` 逻辑

### 4.5 长回复拆多条消息

超过 350 字的回复自动拆成 2 条独立消息，第二条延迟 600-1000ms 后发送。  
像真人一样"分几次发"而非"一篇小作文丢过来"。

**实现位置：** `_splitResponse()` 方法，增加字数判断逻辑

## 五、模块三：情感状态追踪

### 5.1 情绪分类

| 状态 | 触发特征（关键词） |
|------|-------------------|
| 😢 难过 | 分手、失去、想哭、难受、撑不住、心碎、崩溃、抑郁、孤独 |
| 😠 愤怒 | 气死了、凭什么、太过分、受不了、恶心、傻逼、无语、火大 |
| 😰 焦虑 | 怎么办、万一、好怕、担心、睡不着、纠结、迷茫、不知道 |
| 😐 平静 | 无强烈情绪词（默认状态） |

### 5.2 检测算法

- 取最近 3 条用户消息
- 每条消息对三类关键词分别计数
- 同类得分 = 该类命中关键词数之和
- 得分最高且 ≥ 2 的类别 → 激活
- 多个类别同分 → 优先级：难过 > 愤怒 > 焦虑 > 平静

### 5.3 注入的 Prompt 指令

| 状态 | 注入文本 |
|------|---------|
| 难过 | `对方现在情绪低落，第一句话先共情和陪伴，不要急着分析和给建议。语气温柔一点。` |
| 愤怒 | `对方现在很生气，先认可TA的情绪——"确实气人""换谁谁不气"——然后再帮TA理性分析。别泼冷水火上浇油。` |
| 焦虑 | `对方现在很焦虑，你给的建议要比平时更具体和确定，少用反问和留白，多用"我觉得你可以…"的明确指引来帮TA稳住。` |
| 平静 | 不注入额外指令 |

### 5.4 实现位置

新增 `EmotionTracker` 对象，在 `buildContext()` 中调用，追加到 System Prompt 末尾。
